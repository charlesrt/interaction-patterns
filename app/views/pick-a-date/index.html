Pick a date

Pattern principals

  • use when the date is not a memorable one, e.g. Date of Birth
  • use quick pick if the date is likely to be a common time before/from e.g. Two week fit notes
  • don't replace manual input

Start now

Assumptions
  • The user will not always know a specific date
  • The user will have a relative mental model on when the date is, eg next friday, or the first month of next month
  • The interaction can be made simplier with a picker
  • Manual input allows for too many errors
